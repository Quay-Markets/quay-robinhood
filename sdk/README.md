# @quay/evm-sdk

Aggregator/router integration kit for the Quay EVM shared-liquidity propAMM.
Zero runtime dependencies (pure `bigint` math); bring your own client
(viem/ethers) for RPC.

Both integration styles are supported — pick one:

## Path A — one view call (simplest)

No SDK required. The venue exposes a non-reverting quote surface:

```solidity
// token addresses + amount in -> amount out (best book for the pair)
(uint256 amountOut, bytes32 bookId) = quay.getAmountOut(tokenIn, tokenOut, amountIn);

// or per book / all books, with full diagnostics + status codes:
QuoteResult memory r  = quay.quoteExactInput(bookId, tokenIn, amountIn);
QuoteResult[] memory  = quay.batchQuoteExactInput(bookIds, tokenIn, amountIn);
QuoteResult memory r  = quay.quoteBestExactInput(tokenIn, tokenOut, amountIn);
```

Quoter views never revert for routine invalid states — they return
`valid=false` plus a `QuoteReason` code (paused, expired, size, liquidity,
oracle, strategy). Discovery: `getBooksForPair(tokenA, tokenB)` or index the
`BookCreated` event.

## Path B — fetch state, compute locally (Solana style)

One `eth_call` per book (or one for N books) returns everything needed:

```solidity
BookStateView memory v = quay.getBookState(bookId);      // or getBookStates(ids)
// { book, quote, oracle, strategyStatus, groupPaused, protocolPaused,
//   inventory0/1, inventoryNonce0/1 }
```

Then price offline with this package:

```ts
import { quoteExactInput, QuoteReason } from '@quay/evm-sdk';

const r = quoteExactInput({
  token0In: true,            // selling token0 for token1
  amountIn: 10n ** 18n,
  nowSec: BigInt(blockTimestamp),
  protocolFeeBps: BigInt(v.book.protocolFeeBps),
  quote: v.quote,            // bigint-mapped QuoteState
  availableOut: v.inventory1,
  strategy: { kind: 'bbo' }, // or solfi / humidifi / bisonfi + config
});
if (r.valid) route(r.amountOut);
```

The math is **bit-exact** against Solidity: `test-vectors.json` is generated by
the Foundry suite (`forge test --match-contract SdkVectorsTest`) directly from
the deployed pipeline, and `pnpm test` replays all vectors through this
package. If the SDK and the chain ever disagree, the vector tests fail.

Strategy configs (SolFi C-constants, HumidiFi spread params, BisonFi ladders)
are read from the strategy module's public getters; the book's
`book.strategyModule` tells you which pricer applies. Config changes emit
events — index them like quote updates.

## Swapping (the router call)

Plain ERC-20 approve + `swapExactInputSingle` — **no Permit2 anywhere**.
Your router holds the input token (or has custody from the previous leg),
approves the venue, and calls:

```ts
import { encodeSwapExactInputSingle } from '@quay/evm-sdk';

const data = encodeSwapExactInputSingle({
  bookId: r.bookId,
  tokenIn, tokenOut,
  amountIn,
  minAmountOut,              // your slippage bound; swap reverts below it
  recipient,
  deadline: 0n,              // 0 disables
  expectedQuoteNonce: 0n,    // optional: pin the exact quote you priced
  expectedInventoryNonceOut: 0n, // optional: guard shared-group races
});
```

Settlement recomputes the quote on-chain with the same pipeline the views use,
so a swap either fills at ≥ `minAmountOut` or reverts — full fill or nothing,
no partial fills, no router whitelist, no price dependence on `msg.sender`,
`tx.origin`, gas price, or `gasleft()`.

## Two rules that differ from Uniswap-style pools

1. **Quotes decay.** A book's price comes from a maker-pushed quote with a
   freshness window (`validUntil`, plus strategy-level gates). Re-quote close
   to submission; `minAmountOut` protects the rest.
2. **Same-group books share fillability.** Books returning the same
   `liquidityGroupId` draw on the same inventory — don't double-count depth
   across them; use `expectedInventoryNonceOut` when splitting.

## Maker tooling

- `encodeUpdateQuote(bookId, quote)` — direct updater push.
- `quoteUpdateTypedData({...})` — EIP-712 payload for
  `signTypedData`; anyone can relay the signature via
  `updateQuoteWithSig` / `batchUpdateQuotesWithSig`.
